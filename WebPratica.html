Perguntas:

a) É possível sobrepor à aparência (estilo) definida em regras CSS anteriores, mais genéricas, por regras mais específicas? Se possível, essa técnica tem nome? Exemplifique

Sim, é possível sobrepor o estilo, definido por regras CSS genérica, para as mais específicas. E essa técnica é dada pelo nome "Override ou Sobreposição" que neste caso, a regra que foi sobreescrita, terá mais potencial, ou seja, será a técnica executada num código html.

b) Caso definíssemos uma regra CSS mais geral para cabeçalhos de nível 1, 2 e 3, seguida de outra somente para o nível 1, como seria o comportamento do navegador? Descreva os vários cenários possíveis e o comportamento do navegador em cada uma delas

Quando define uma regra geral para os níveis 1, 2 e 3, o comportamento do navegador será obedecer o que lhe foi pedido, ou seja, ele aplicará a mesma regra a todos os níveis. Mas, quando além do que foi definido para os três níveis estiver uma outra definição apenas para o nível 1, significa dizer que o navegador, irá aplicar ao nível 1 essa última regra. E aos níveis 2 e 3 continuarão as mesmas regras de "antes" (a primeira regra aplicada).

c) É sempre interessante otimizar o código CSS? Cite algumas vantagens

Sim, pois otimizando um código html, deixará o código mais "limpo", ou seja, o código não ficará muito extenso. E também, ao deixar o código mais "limpo", estará economizando armazenamento.

d) Defina herança em CSS. Utilize, também, imagens para ilustrar tais definições.

Herança no css, é como uma herança numa classe java. A diferença é que, na classe Java são herdados métodos e atributos, e no CSS é herdado as "regras" de um "pai". Um exemplo: Colocar regras no body: Tudo o que for adicionado no Body (CSS), todo o conteúdo que tiver dentro do body (html), vai herdar essa regra.
